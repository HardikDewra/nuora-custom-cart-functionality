Section 01

What We're Building

Two interconnected components that share the same JS module and data layer. Both must work on theme pages, Replo pages, and any page where a product gets added to cart.

Component A
Cart Drawer

Slide-out panel triggered by cart icon click. Slides in from the right. Contains line items, cross-sells, free shipping bar, bundle upsells, and checkout CTA. 420px on desktop, 100% on mobile.

Component B
Custom /cart Page

Full-page cart at /pages/cart. Default /cart redirects here. Same features as drawer but expanded with room for cross-sell cards, trust badges, and subscription details.

Goal
Increase AOV

Upsells and cross-sells directly in the cart experience. Every add-to-cart is an AOV opportunity.

Coverage
Sitewide

Theme pages (homepage, collections, blog) + Replo pages (PDPs, presell pages, listicles). One script covers everything.

Stack
Vanilla JS

Single JS module, no framework deps. Uses Shopify Cart AJAX API. Loaded via theme.liquid before </body>.

Injection Strategy

Theme Pages

Load nuora-cart.js via theme.liquid before </body>. Script intercepts all add-to-cart form submissions, injects drawer HTML into DOM, listens for cart:open event, and hijacks cart icon clicks.

Replo Pages

Replo pages load inside theme layout, so theme.liquid injection covers them automatically. Replo's add-to-cart must fire a cart:refresh custom event that the cart module listens to.

Event Bridge (Theme + Replo) 
// After any add-to-cart action (theme or Replo), fire this:
document.dispatchEvent(new CustomEvent('cart:refresh'));

// The cart module listens:
document.addEventListener('cart:refresh', async () => {
  const cart = await fetch(window.Shopify.routes.root + 'cart.js')
    .then(r => r.json());
  updateCartState(cart);
  renderCartDrawer(cart);
  openDrawer();
});

Section 04

AOV Features

Four AOV tactics built into the cart experience. Each one has its own logic and display rules.

Feature 01
Free Shipping Progress Bar

Free shipping when ANY variant in cart is a 2+ pack/bottle (e.g. 2 Packs, 3 Bottles). Two separate 1-unit products do NOT qualify. Only a multi-pack variant triggers free shipping. Not based on dollar amount.

Shipping Logic 
// FREE SHIPPING: only if ANY variant is a 2+ pack/bottle.
// Two separate 1-unit products do NOT qualify.
// Examples:
//   1 Pack gummies + 1 Bottle capsules = NOT free
//   2 Packs gummies + 1 Bottle capsules = FREE (2-pack variant)
//   3 Bottles capsules only = FREE (3-bottle variant)

function hasMultiPackVariant(cartItems) {
  return cartItems.some(item => {
    const variantTitle = (item.variant_title || '').toLowerCase();
    const match = variantTitle.match(/(\d+)\s*(bottle|pack|capsule)/);
    const unitsPerVariant = match ? parseInt(match[1]) : 1;
    return unitsPerVariant >= 2;
  });
}

function getShippingProgress(cartItems) {
  if (hasMultiPackVariant(cartItems)) {
    return {
      qualified: true,
      percentage: 100,
      message: "You've unlocked FREE shipping!",
    };
  }

  return {
    qualified: false,
    percentage: 50,
    message: 'Upgrade to 2+ packs for FREE shipping!',
  };
}

Feature 03
Bundle Upsell (Upgrade to Higher Pack)

When a 1-bottle variant is in cart, prompt to upgrade to 3-pack. Preserves selling plan on upgrade. Removes old item, adds new variant.

Upgrade Action 
async function upgradeToPack(currentItemKey, newVariantId, sellingPlanId) {
  // Remove current 1-pack item
  await fetch(window.Shopify.routes.root + 'cart/change.js', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ id: currentItemKey, quantity: 0 }),
  });

  // Add upgraded 3-pack variant
  const addBody = { id: newVariantId, quantity: 1 };
  if (sellingPlanId) addBody.selling_plan = sellingPlanId;

  await fetch(window.Shopify.routes.root + 'cart/add.js', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ items: [addBody] }),
  });

  document.dispatchEvent(new CustomEvent('cart:refresh'));
}
Feature 04
Quantity Upsell

Two options under consideration:

Option A

Increment qty normally and show tiered savings messaging ("Buy 2 more, save X%").

Option B (Recommended)

Hide qty adjuster entirely, only show bundle upgrade prompts. Prevents SKU mapping issues with SKIO/Ka-ching where qty 3 of a 1-bottle variant ships wrong.

Section 07

SKIO / Selling Plan Gotchas

Based on past issues with Ka-ching and SKIO. Read these carefully before implementation.

Gotcha #1 - Critical
Use line item key, NOT variant_id for cart/change.js

Multiple line items can share the same variant_id if they have different selling plans. Always use the unique key property from the cart item.

Gotcha #2
Selling plan IDs are per selling plan group

Duplicated products for A/B tests each have their own selling plan group with different IDs. Cross-sell logic needs to know which variant + selling_plan combo to use.

Gotcha #3 - Critical
Don't mix Ka-ching bundles with cart AJAX

If Ka-ching is still active on any products, its bundle logic may conflict with direct cart API calls. Confirm with Elliot which products are fully on SKIO selling plans only.

Gotcha #4
cart/clear before subscription checkout (legacy pattern)

The old buy boxes used cart/clear -> cart/add -> /checkout URLs. The new cart module replaces this. Subscriptions added via Cart AJAX API with selling_plan flow through checkout normally.

Gotcha #5
Currency handling

Cart API returns prices in presentment currency. Use Shopify.currency.active for the currency code. Don't hardcode $.

Section 08

Theme Integration Guide

Step-by-step for Tomide. The Shopify theme already has a built-in cart drawer - our custom drawer replaces it, not stacks on top.

Step 1 - Critical
Disable the theme's built-in cart drawer

The theme has its own slide-out cart. If you add ours without disabling theirs, you'll get two drawers fighting each other. Check theme settings for a "Cart type" option and set it to "Page" (not "Drawer"). If there's no setting, find the theme's cart drawer snippet (usually cart-drawer.liquid or similar) and comment out the include in theme.liquid.

Step 2 - Critical
Neutralize the theme's cart icon JS

The theme's cart icon is wired to open THEIR drawer via their own JS. Our script hijacks a[href="/cart"] clicks with preventDefault, but the theme's JS might fire first. After disabling their drawer (Step 1), their JS should have nothing to open. If the icon still behaves weird, find their cart icon element and add data-cart-trigger attribute so our code picks it up cleanly.

Step 3
Upload our files to the theme

Upload nuora-cart-drawer.liquid to /snippets/, nuora-cart-drawer.css and nuora-cart-drawer.js to /assets/. Then add the CSS tag in <head> and the Liquid render + JS tag before </body> in theme.liquid. See the README for exact code.

Step 4
Theme pages: auto-detected

Our script auto-intercepts all form[action="/cart/add"] submissions on theme pages (PDPs, collections, etc). No extra work needed. The drawer opens automatically after add-to-cart.

Step 5 - Critical
Replo pages: add one line of custom code

Replo's ATC buttons are custom components, not standard Shopify forms. Our script can't auto-detect them. After each Replo add-to-cart action, fire this event in a Replo custom code block:

Replo Custom Code (after ATC) 
// Add this in Replo's custom code block, after the add-to-cart action
document.dispatchEvent(new CustomEvent('cart:refresh'));

// This tells our drawer: "something was added, refresh cart data and open"
// Works from any Replo page - PDPs, presell pages, listicles, etc.
Testing After Integration

After completing all steps, verify: (1) Only ONE drawer appears when cart icon is clicked. (2) Theme pages ATC opens our drawer automatically. (3) Replo pages ATC opens our drawer via the custom event. (4) The /cart URL redirects to /pages/cart (if using Replo cart page). If two drawers appear, Step 1 wasn't fully done - find and disable the theme's drawer.

Section 12

Cart State Manager (Core Module)

The single source of truth. Both the drawer and /cart page subscribe to this. Full implementation reference below.

nuora-cart.js - Core Module 
const NuoraCart = (() => {
  let state = {
    cart: null,
    isDrawerOpen: false,
    crossSellOffer: null,
    bundleUpsell: null,
    shippingProgress: null,
  };

  const listeners = [];

  function subscribe(fn) { listeners.push(fn); }
  function notify() { listeners.forEach(fn => fn(state)); }

  async function refresh() {
    const root = window.Shopify.routes.root;
    state.cart = await fetch(root + 'cart.js').then(r => r.json());
    state.crossSellOffer = await getCrossSellOffer(state.cart.items);
    state.bundleUpsell = getBundleUpsell(state.cart.items);
    state.shippingProgress = getShippingProgress(state.cart.total_price);
    notify();
    updateCartBadge(state.cart.item_count);
  }

  async function addItem(variantId, quantity = 1, sellingPlan = null) {
    const body = { items: [{ id: variantId, quantity }] };
    if (sellingPlan) body.items[0].selling_plan = sellingPlan;
    await fetch(window.Shopify.routes.root + 'cart/add.js', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(body),
    });
    await refresh();
  }

  async function updateItem(lineItemKey, quantity) {
    await fetch(window.Shopify.routes.root + 'cart/change.js', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ id: lineItemKey, quantity }),
    });
    await refresh();
  }

  async function removeItem(lineItemKey) {
    await updateItem(lineItemKey, 0);
  }

  function openDrawer() {
    state.isDrawerOpen = true;
    document.body.style.overflow = 'hidden';
    notify();
  }

  function closeDrawer() {
    state.isDrawerOpen = false;
    document.body.style.overflow = '';
    notify();
  }

  function updateCartBadge(count) {
    document.querySelectorAll('[data-cart-count]').forEach(el => {
      el.textContent = count;
      el.style.display = count > 0 ? '' : 'none';
    });
  }

  async function init() {
    await refresh();

    // Intercept theme add-to-cart forms
    document.addEventListener('submit', async (e) => {
      const form = e.target.closest('form[action="/cart/add"]');
      if (!form) return;
      e.preventDefault();
      const formData = new FormData(form);
      const id = formData.get('id');
      const qty = formData.get('quantity') || 1;
      const sellingPlan = formData.get('selling_plan');
      await addItem(parseInt(id), parseInt(qty),
        sellingPlan ? parseInt(sellingPlan) : null);
      openDrawer();
    });

    // Listen for custom events
    document.addEventListener('cart:refresh', refresh);
    document.addEventListener('cart:open', openDrawer);
    document.addEventListener('cart:close', closeDrawer);

    // Hijack cart icon clicks
    document.querySelectorAll(
      'a[href="/cart"], .cart-icon, [data-cart-trigger]'
    ).forEach(el => {
      el.addEventListener('click', (e) => {
        e.preventDefault();
        openDrawer();
      });
    });
  }

  return {
    init, subscribe, refresh,
    addItem, updateItem, removeItem,
    openDrawer, closeDrawer,
    getState: () => state
  };
})();

// Auto-init
if (document.readyState === 'loading') {
  document.addEventListener('DOMContentLoaded', NuoraCart.init);
} else {
  NuoraCart.init();
}
